# 智能多维表格功能优化计划

根据您的反馈，我们将对“表单助手”进行深度优化，重点解决初始化体验、布局控制、知识库路径挂载及筛选功能的问题。

## 1. 优化初始化体验 (Sheet-like Default)
当前问题：默认展示项目管理模板，且列宽布局异常导致初始行过长。
解决方案：
- **默认列重置**：初始化时默认生成 8 列（A-H），不再预设具体字段。
- **默认行重置**：初始化生成 20 行空白行，提供标准的电子表格体验。
- **布局调整**：修复初始加载时的列宽分配问题，确保 A-H 列均匀分布或适配默认宽度。

## 2. 严格固定单元格宽高 (Strict Layout)
当前问题：单元格宽高未被严格锁定，内容过多时会撑开。
解决方案：
- **强制表格布局**：使用 `table-layout: fixed` 强制浏览器遵循列宽定义。
- **严格样式锁定**：在 `td` 和 `th` 上同时应用 `width`, `min-width`, `max-width`，确保尺寸绝对固定。
- **内容截断优化**：确保单元格内部 `div` 拥有 `w-full h-full overflow-hidden truncate` 属性，配合已实现的悬停预览功能，完美处理长文本。

## 3. 修复知识库挂载路径 (Fix KB Integration)
当前问题：右键导入时显示的文件夹与实际知识库不一致。
解决方案：
- **路径溯源**：经排查，`warehousePath` 已从 `App.tsx` 透传至组件。我们将优化 `loadFiles` 逻辑，确保在组件挂载及 `fileImportModal` 打开时，准确读取 `warehousePath`。
- **异常处理**：如果 `warehousePath` 为空或读取失败，将显示明确的“请在设置中配置知识库路径”提示，而不是加载模拟数据（Mock Data），避免误导。
- **API 调用修正**：确保 `window.electronAPI.fs.readDirectory` 被正确调用。

## 4. 修复筛选功能 (Fix Filtering)
当前问题：列头右键筛选功能无效。
解决方案：
- **交互优化**：当前使用 `prompt` 可能被拦截或体验不佳。我们将改为在右键菜单中直接提供一个**行内输入框**，用户输入关键词后按回车即可筛选。
- **逻辑增强**：确保筛选逻辑在过滤前剔除 HTML 标签（`stripTags`），实现对富文本内容的精准纯文本匹配。

## 实施步骤
1.  **重构状态初始化**：修改 `useState` 默认值，生成 A-H 列和空白行。
2.  **样式加固**：在 CSS 类中强制应用 `table-fixed` 和尺寸锁定。
3.  **路径逻辑修正**：移除开发环境的 Mock 路径回退，增加路径有效性检查。
4.  **筛选UI改进**：在 ContextMenu 中实现自定义输入组件。
