# 多维表格“逻辑闭环”功能升级计划

根据您的创意，我们将把多维表格从“数据记录表”升级为“AI 工作流画布”。核心目标是实现**多态内容渲染**和**可编排的 AI 逻辑闭环**。

## 1. 核心架构升级 (Core Architecture)
### A. 多态单元格 (Polymorphic Smart Cell)
不再将单元格视为纯文本容器，而是根据内容特征自动识别并渲染为不同形态：
-   **代码块 (Code)**：识别 Markdown 代码块（```json ...```），提供语法高亮和折叠功能。
-   **JSON 对象 (Data)**：识别 JSON 字符串，渲染为可交互的树状结构。
-   **加载状态 (Status)**：为正在运行 AI 任务的单元格提供明确的 Loading/Spinner 状态。
-   **富文本 (Rich Text)**：继续支持 Markdown 渲染（加粗、列表、链接）。

### B. 逻辑闭环工作流 (Logic Loop)
-   **自定义 Prompt 模板**：在列配置中增加“Prompt 模板”字段。用户可以引用其他列的内容（如 `{{col-A}}`），而不仅仅是把所有列都丢给 AI。
-   **单行执行流 (Row Execution)**：增加“运行此行”功能，按照列的顺序触发 AI 任务，实现 A -> B -> C 的数据加工流水线。

## 2. 功能实现步骤
1.  **新建 `SmartCell` 组件**：
    -   封装内容渲染逻辑，自动检测 Markdown 代码块和 JSON。
    -   集成 `react-syntax-highlighter`（如果无法引入新库，则使用简易的样式高亮）。
2.  **升级 `Column` 数据结构**：
    -   增加 `promptTemplate` 字段。
3.  **升级 `handleRunColumnAgent` 逻辑**：
    -   支持解析 `{{col-ID}}` 占位符，构建精准的 Context。
4.  **UI 交互增强**：
    -   在列头菜单中增加“运行此列”按钮（现有功能的优化版）。
    -   在行头增加“运行工作流”按钮（依次运行该行所有配置了 AI 的列）。

## 3. 预期效果
用户可以在 A 列输入一段原始数据（如 JSON），B 列配置“提取关键信息”的 Prompt，C 列配置“生成 Python 处理代码”的 Prompt。点击运行后，表格自动完成 Data -> Info -> Code 的转化，且 C 列直接显示高亮代码。
