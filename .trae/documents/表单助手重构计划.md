# 表单助手 (Forms Assistant) 重构计划

根据您的需求，我们将把现有的 `MultidimensionalTable` 拆分并升级为三个独立的功能模块，分别对应**可视化**、**轻量级Excel**和**Agent工作流**。

## 1. 架构重组 (Architecture Refactoring)
我们将把 `MultidimensionalTable.tsx` 拆分为一个主入口容器 `FormsAssistant.tsx` 和三个子组件：
1.  **`DataVisualizer.tsx`** (新功能)：负责 HTML 交互式可视化生成。
2.  **`SimpleSpreadsheet.tsx`** (新功能)：支持 AI 函数的精简 Excel。
3.  **`AgentWorkflowTable.tsx`** (原 MultidimensionalTable)：专注于 AI 工作流编排。

## 2. 模块详细设计

### 模块一：表单可视化 (Data Visualization)
*   **输入**：支持用户粘贴 JSON/CSV 数据，或选择“从其他模块导入”。
*   **处理**：调用 LLM (Gemini) 分析数据结构，生成一段包含 Chart.js/ECharts 的 HTML 代码。
*   **输出**：
    *   **实时预览**：在一个 iframe 中渲染生成的 HTML。
    *   **导出/分享**：提供 "Save as HTML" 按钮，下载单文件 HTML，用户可直接发给别人用浏览器打开。

### 模块二：AI 智能表格 (Simple AI Spreadsheet)
*   **定位**：常规 CRUD 操作，但操作逻辑 AI 化。
*   **核心功能**：
    *   **基础操作**：增删改查行/列。
    *   **AI 函数栏**：顶部提供一个自然语言输入框（代替 Excel 公式栏）。
        *   用户输入：“计算 C 列的总和，如果超过 1000 就标红”。
        *   AI 执行：解析指令，直接修改数据或应用样式。
*   **实现**：基于原 `MultidimensionalTable` 的基础数据结构，但移除复杂的 Agent 配置，专注于**单次指令交互**。

### 模块三：多维表格 (Agent Workflow Table)
*   **定位**：保留原有的“列即 Agent”和“流水线”逻辑。
*   **清理**：移除与 Agent 工作流无关的功能（如刚才提到的普通增删改查 UI），使其更专注于**自动化配置**和**批量运行**。

## 3. 实施步骤
1.  **创建容器组件**：新建 `components/FormsAssistant.tsx`，实现顶部 Tab 切换（可视化 / 智能表格 / 多维表格）。
2.  **拆分与实现**：
    *   将原 `MultidimensionalTable` 重命名为 `AgentWorkflowTable`。
    *   新建 `SimpleSpreadsheet`，复用基础 Table UI，但增加自然语言指令栏。
    *   新建 `DataVisualizer`，实现“数据 -> Prompt -> HTML -> Iframe”的渲染链路。
3.  **集成**：在 `AIAgentWorkspace` 中，当选择 `Forms` 房间时，渲染新的 `FormsAssistant` 容器。

## 4. 预期效果
用户进入“表单助手”后，可以看到三个清晰的选项卡：
*   **可视化**：把枯燥的数据变成可交互的网页仪表盘。
*   **智能表格**：用说话代替 Excel 公式的日常数据处理。
*   **多维表格**：配置复杂的 AI 流水线，批量生产内容。
