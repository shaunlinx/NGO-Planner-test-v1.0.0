# 高级工作流模版实现计划

您提出的两个工作流非常具有代表性，分别覆盖了“内容生产发布”和“数据采集清洗”两大核心场景。为了在当前的多维表格中实现这两个高阶功能，我将通过**预置模版 (Templates)** + **增强型 Agent 模拟** 来落地。

## 1. 功能架构升级

### A. 模版加载系统 (Template System)
在工具栏增加“场景模版”菜单，包含您定制的两个工作流。加载模版将自动配置好所有列的名称、类型、关联的 AI 专家以及 Prompt 模板。

### B. 多媒体与模拟能力增强
1.  **视觉助手支持 (Visual Support)**：升级 `SmartCell`，使其支持渲染图片（Markdown 格式 `![img](url)`）。当“视觉助手”工作时，将返回（模拟的）图片链接，直接在表格中显示海报。
2.  **外部动作模拟 (Action Simulation)**：
    -   **社群助手**：由于无法直接调用微信/小红书闭源 API，Agent 将返回“✅ 已模拟打开登录窗口... 已存入草稿箱”的状态日志。
    -   **数字田野调查**：Agent 将模拟“截图-拼接-OCR-清洗”的全过程，并返回清洗后的结构化文本。

## 2. 详细模版设计

### 模版一：全媒体内容生产流水线 (Content & Publishing Pipeline)
此模版将包含以下自动化列：
1.  **A列：选题/灵感 (Input)**：输入如“春季公益植树活动”。
2.  **B列：文案生成 (Copywriter)**：引用 A 列，生成适配社媒的文案。
3.  **C列：视觉海报 (Designer)**：引用 A 列，生成海报图片（将使用占位符图片模拟）。
4.  **D列：图文排版 (Layout)**：引用 B+C 列，生成手机端预览格式（HTML/Markdown）。
5.  **E列：一键发布 (Publisher)**：引用 D 列，执行模拟发布动作，返回操作日志。

### 模版二：数字田野调查自动化 (Digital Field Research)
此模版专注于从 URL 到数据的转化：
1.  **A列：目标链接 (Link)**：输入小红书/公众号链接。
2.  **B列：采集状态 (Status)**：显示“截图拼接中... OCR识别中...”。
3.  **C列：清洗结果 (Result)**：展示经过 OCR 和清洗后的核心内容摘要。
4.  **D列：结构化数据 (JSON)**：将非结构化内容转化为表格数据（如：发布时间、作者、正文、评论数）。

## 3. 实施步骤
1.  **定义模版数据结构**：在代码中硬编码这两个模版的配置（列定义、Prompt、示例数据）。
2.  **升级 UI**：在表格顶部添加“模版库”下拉菜单。
3.  **升级渲染器**：让单元格支持图片显示。
4.  **增强 Mock 逻辑**：在 `runAgentTask` 中增加针对“视觉助手”（返回图片）和“田野调查”（返回模拟清洗数据）的特殊处理逻辑。
