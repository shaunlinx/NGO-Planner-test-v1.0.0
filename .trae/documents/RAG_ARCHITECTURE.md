# RAG 架构与功能技术白皮书 (v3.0)

## 1. 核心架构概览

本系统采用 **本地优先 (Local-First)** 的混合 RAG 架构，结合了传统的关键词检索与现代的向量语义检索，并辅以重排序 (Rerank) 和 AI 智能增强，旨在为个人知识库提供高精度、低延迟的问答体验。

### 1.1 数据流全景
1.  **解析 (Ingestion)**: 多模态文件 -> 智能清洗/布局恢复 -> 语义切块 (Chunking) -> 上下文增强 (Enrichment)。
2.  **索引 (Indexing)**: 双路存储 -> LanceDB (向量索引) + SQLite (元数据与统计)。
3.  **检索 (Retrieval)**: 混合检索 (Hybrid Search) -> 向量相似度 + BM25 关键词 -> 候选集融合。
4.  **排序 (Ranking)**: 候选集 -> 权重干预 (Weighting) -> 交叉编码器重排序 (Cross-Encoder Rerank) -> Top-K。
5.  **生成 (Generation)**: 检索上下文 -> 提示词工程 -> LLM 回答。

## 2. 关键模块详解

### A. 智能文件处理 (`FileProcessor`)
*   **PDF 智能解析**: 放弃了传统的 `pdftotext`，基于 `pdf.js` 自研了流式解析器。
    *   **布局恢复**: 通过坐标 (X, Y) 排序重建阅读顺序，智能识别多栏排版。
    *   **CJK 优化**: 针对中日韩字符间距进行特殊处理，杜绝“中 文 乱 加 空 格”。
    *   **标题识别**: 基于字体大小动态识别 H1/H2，保留文档结构信息。
*   **多模态支持**: 
    *   Word/PPT/Excel: 结构化提取。
    *   Audio/Video: 集成 OpenAI Whisper 进行语音转文字。
    *   Images: 集成 Tesseract OCR 进行图片文字提取。

### B. 语义切片与增强 (`Chunking Strategy`)
*   **递归切分**: 采用 `RecursiveCharacterTextSplitter` 策略，优先按段落、句子切分，最后按字符切分，保留语义完整性。
*   **上下文增强 (Context Enrichment)**: 
    *   **Agentic Ingestion**: 在入库时，可选调用 LLM 为每个切片生成“一句话上下文摘要”（如：“此段落涉及2024年Q3财务报表”），并将其拼接到向量内容中。这使得即使用户问题不包含具体关键词，也能通过语义匹配到相关切片。

### C. 混合检索与重排序 (`RAGEngine`)
*   **双路召回 (Hybrid Search)**:
    *   **路一 (Dense)**: 使用 Embedding 模型（如 `text-embedding-3-small`）捕捉语义相似度。
    *   **路二 (Sparse)**: 模拟 BM25 算法进行关键词匹配，弥补向量检索对专有名词（如项目代号、人名）不敏感的缺陷。
*   **候选集融合**: 将两路召回的 Top-50 结果去重合并。
*   **权重干预**: 引入用户自定义权重 (`weight_factor`)，允许用户手动提高重要文件的被引概率。
*   **重排序 (Rerank)**: 
    *   集成 **BGE-M3 / BGE-Reranker** (通过 `@xenova/transformers` 本地运行)。
    *   对候选集进行精细打分，剔除“语义相关但逻辑无关”的噪音，显著提升 Top-3 准确率。

### D. 知识库生命周期管理 (KLM)
*   **可视化管理**: 提供文件级和切片级的全景视图。
*   **切片手术刀**: 支持查看、搜索、编辑、删除任意切片。
*   **智能运维**:
    *   **AI 批量清洗**: 自动识别并删除包含特定特征（如“机密”、“页眉”）的切片。
    *   **AI 批量润色**: 调用 LLM 对切片内容进行重写（如口语转书面语），并自动触发向量重算 (Re-embedding)。
    *   **热度监控**: 追踪每个文件的引用次数，识别“僵尸文档”和“核心文档”。

## 3. 技术栈清单

| 模块 | 技术选型 | 优势 |
| :--- | :--- | :--- |
| **应用框架** | Electron + React | 跨平台桌面端，高性能 UI |
| **向量数据库** | **LanceDB** (Embedded) | 无需 Docker，进程内运行，极速读写，支持 SQL 查询 |
| **关系数据库** | SQLite | 存储元数据、标签、统计信息 |
| **模型推理** | ONNX Runtime (本地) + API | 本地运行 Reranker，云端运行 LLM/Embedding |
| **PDF 引擎** | PDF.js (Legacy Build) | 纯 JS 实现，无原生依赖，支持 CMap 中文解析 |

## 4. 差异化优势

1.  **隐私与离线能力**: 核心索引和重排序在本地完成，不仅保护隐私，且无网络延迟。
2.  **可解释性与可控性**: 相比于黑盒的 LangChain 封装，本系统暴露了从切片到排序的所有中间环节，允许用户手动干预（加权、改词）。
3.  **闭环优化**: 从“入库”到“使用”再到“反馈修改”，形成了完整的数据质量闭环。用户在使用中发现回答不准，可以直接定位到切片进行修改，立竿见影。
