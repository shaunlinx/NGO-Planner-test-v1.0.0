I will implement a "Deep Thinking Mode" (深度思考模式) to solve the "fragmented answer" problem. This introduces an **Agentic RAG** architecture:

1.  **Planner Agent (The "Brain")**:
    *   Instead of searching immediately, the AI first analyzes the user's complex query (e.g., "Summarize the project") and generates a **Logical Framework/Outline** (e.g., "1. Background, 2. Key Actions, 3. Results, 4. Future Plans").
    *   This ensures the final answer has a coherent structure before any data is retrieved.

2.  **Guided Retrieval (The "Worker")**:
    *   The system will generate specific, targeted search queries for *each section* of the outline.
    *   *Example*: Instead of one generic search for "Project Summary", it searches "Project Background" -> "Project Results" -> "Project Risks" separately.
    *   This ensures "irrelevant high-ranking chunks" don't crowd out specific, necessary details for other sections.

3.  **Structured Synthesis (The "Writer")**:
    *   The final answer is generated by filling in the framework with the specific evidence found for each section, ensuring a comprehensive and logical output.

**Implementation Steps:**

1.  **Backend Service (`geminiService.ts`)**:
    *   Add `generateAnswerFramework`: To create the research outline.
    *   Add `generateDeepSynthesis`: To write the final answer based on the structured context.
2.  **Frontend (`KnowledgeBase.tsx`)**:
    *   Add a "Deep Thinking" (深度思考) toggle switch.
    *   Implement the multi-step orchestration:
        *   **Step 1**: "Planning..." (Call Framework API)
        *   **Step 2**: "Researching..." (Parallel calls to `electronAPI.knowledge.query` for each sub-topic)
        *   **Step 3**: "Writing..." (Synthesize final answer)
    *   Add visual progress indicators for these steps.

This moves the system from "Search -> Answer" to "Plan -> Research -> Answer", significantly improving quality for complex tasks.
