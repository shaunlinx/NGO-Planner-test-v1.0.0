# 修复与优化计划

根据您的反馈，我制定了以下修复数据丢失问题并优化软件体积的计划。

## 1. 修复数据丢失问题 (Data Loss Fix)
**原因分析**: 您遇到的“数据丢失”很可能是由于应用名称（Product Name）变更导致的数据存储目录变化。旧版本可能使用了 `ngo-planner-desktop` 或 `NGO Planner` 作为目录名，而新版本（公益人年历）使用了新的目录，导致系统无法加载旧数据。

**解决方案**:
在 `databaseManager.js` 中增加**自动数据迁移逻辑**。
- 当检测到当前数据库为空（新安装）时，自动尝试查找旧版数据目录（如 `~/Library/Application Support/ngo-planner-desktop`）。
- 如果找到旧数据，自动将其复制到新目录，确保用户无感过渡。

## 2. 软件体积优化 (Size Optimization)
**原因分析**: 安装包体积过大（~2GB）主要是因为内置了多个大型 AI 模型（`bge-m3` 约 600MB, `bge-reranker` 约 300MB）。

**解决方案**:
- **删除冗余模型**: 移除 `resources/models/Xenova/bge-m3` 和 `bge-reranker-base`。
- **保留轻量模型**: 仅保留 `all-MiniLM-L6-v2`（约 50MB）。
- **代码验证**: `embedding.js` 中已包含自动降级逻辑（`Fallback to all-MiniLM-L6-v2`），删除文件后软件仍能正常运行，仅使用轻量模型。

## 3. 依赖与打包
- **确认修复**: 再次确认 `apache-arrow` 已添加到依赖中。
- **执行打包**: 运行 `npm run dist:x64` 生成最终的优化版安装包。
