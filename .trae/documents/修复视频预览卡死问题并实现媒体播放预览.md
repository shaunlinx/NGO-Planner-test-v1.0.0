# 修复视频文件预览导致软件卡死的问题

## 问题诊断
**现象**：点击 `.MOV` 视频文件预览时，软件界面完全卡死，且无新日志产生。
**原因**：
1.  **主进程阻塞**：`electron/main.js` 中的 `fs-read-file-preview` 方法没有专门处理视频格式。它将 `.MOV` 文件视为普通文本文件，尝试使用 `fs.readFileSync(path, 'utf8')` 一次性将整个巨大的视频文件加载到内存字符串中。
2.  **资源耗尽**：对于几十兆甚至几百兆的视频，这会瞬间耗尽内存并阻塞 Node.js 事件循环，导致应用无响应（Freeze）。

## 修复方案

### 1. 后端修复 (electron/main.js)
修改 `fs-read-file-preview` IPC 处理器：
- **增加媒体格式识别**：显式检测 `.mp3`, `.wav`, `.mp4`, `.mov`, `.webm` 等扩展名。
- **避免读取文件内容**：对于媒体文件，**不**读取其二进制内容，而是直接返回 `{ type: 'media', data: filePath }`。让前端通过文件路径直接播放，而不是通过 IPC 传输大文件数据。

### 2. 前端增强 (components/IndexManager.tsx)
升级预览组件以支持 'media' 类型：
- **媒体播放器**：根据文件类型渲染 `<video>` 或 `<audio>` 标签。
- **图文对照**：沿用之前图片的“左图右文”或“上图下文”布局，左侧/上方显示播放器，右侧/下方显示从知识库中检索到的**转录文本**（Chunk Text）。
- **交互优化**：确保用户既能播放音视频，又能看到由本地 Python 脚本生成的带时间戳的文本。

## 验证计划
1.  重启应用以应用主进程更改。
2.  在知识库中点击该 `.MOV` 文件进行预览。
3.  确认软件不再卡死。
4.  确认预览区显示视频播放器，并能看到转录的文字内容。
