# 多维表格“生产级”功能重构计划

您的反馈非常准确。当前功能虽然有了“形”（UI和基础逻辑），但缺乏“神”（稳健性、易用性和深度集成）。要实现您所设想的“AI 工作流画布”，我们需要从底层架构到交互细节进行全面重构。

这是一个将 Demo 级产品提升为 Production 级产品的系统性工程。

## 1. 核心痛点分析
- **数据不可靠**：缺乏强类型约束（数字列能填文字，日期列无选择器），导致 AI 处理时输入垃圾数据。
- **操作效率低**：缺乏键盘导航（Tab/Enter 切换）、撤销重做、批量选区操作。
- **AI 割裂感**：AI 任务是离散的，缺乏任务队列、并发控制和错误重试机制。用户不敢大规模运行。
- **视觉噪音**：JSON/代码块虽然能渲染，但缺乏折叠、全屏查看等深度交互。

## 2. 功能策划与升级方案

### A. 基础体验重构 (The Foundation)
1.  **强类型系统 (Type System)**：
    -   **实现**：为 `Column` 定义严格的 `type`（Text, Number, Date, Select, Tag）。
    -   **交互**：根据类型渲染不同的编辑器（日期选择器、下拉菜单、数字步进器）。
    -   **验证**：输入时进行校验，非法值标红或拒绝录入。
2.  **键盘导航与选区 (Grid Navigation)**：
    -   **实现**：支持方向键移动焦点，Enter 进入编辑，Tab 切换下列。
    -   **交互**：实现类似 Excel 的蓝色焦点框，支持 Shift 多选区域。
3.  **状态管理增强**：
    -   **撤销/重做 (Undo/Redo)**：引入 `useHistory` hook，记录操作栈。
    -   **持久化**：确保列宽、排序、筛选配置在刷新后保留。

### B. AI 工作流引擎 (The AI Engine)
1.  **任务队列系统 (Task Queue)**：
    -   **痛点**：当前循环 `await` 会卡死浏览器。
    -   **方案**：建立后台任务队列。用户点击“运行列”后，任务进入 `Pending` 状态，UI 显示进度条（如 "Processing 5/20..."）。支持暂停/取消。
2.  **智能重试与纠错**：
    -   **功能**：当 AI 返回格式错误（如 JSON 解析失败）时，自动进行一次“自我修正”重试，或标记为 Error 状态供人工干预。
3.  **依赖执行 (DAG Execution)**：
    -   **功能**：如果 C 列依赖 B 列，用户修改 A 列后，自动标记 B、C 为“过期（Stale）”状态，提示用户重新运行，或配置为自动级联更新。

### C. 深度交互组件 (Rich Components)
1.  **高级多态单元格**：
    -   **JSON 交互**：不仅是展示，支持点击 Key/Value 进行简单的修改。
    -   **代码编辑器**：集成轻量级 Monaco Editor 或类似组件，支持语法高亮和简单的代码编辑。
    -   **文件/图片预览**：支持直接在单元格内预览图片缩略图。
2.  **侧边栏详情页 (Side Peek)**：
    -   **功能**：点击行首“展开”按钮，在右侧抽屉中以表单形式查看该行所有数据。适合处理长文本阅读和编辑。

## 3. 实施路线图 (Roadmap)

我们将分两步走：

**第一阶段：坚实基础（本次执行）**
1.  **类型化编辑器**：引入 DatePicker 和 Select 组件，落实列类型约束。
2.  **键盘导航**：实现基础的方向键导航和 Enter 编辑模式，脱离纯鼠标操作。
3.  **任务状态可视化**：为 AI 单元格增加明确的状态机（Idle -> Queued -> Running -> Success/Error），并提供重试按钮。

**第二阶段：高级特性（后续迭代）**
-   虚拟滚动（解决大数据性能）。
-   撤销/重做栈。
-   复杂的 DAG 依赖自动更新。

本次我们将集中精力完成 **第一阶段**，让表格变得“好用”且“可控”。
